Sound, Visuals + Code

Processing Experiments

As I design more and more for interactive experiences, I wanted to have a better understanding of the code behind the apps and web pages we use every day. To get exposed directly to the coding side of interactive development, I enrolled in a Coursera class titled Creative Programming. Here are a few of the projects I worked on.

Project 1 - Audio Mixer

A relatively simple application that starts and stops playback and controls the speed of the audio and visuals.

It was an interesting challenge to create an interesting application with the constraints of how little I knew about coding at this point in the class. I chose and edited the video and soundfiles to establish an atmosphere and mood.

Processing Project 1 - Audio Mixer from Ryan Hansz on Vimeo.

Project 2 - Music Visualizer

Controlling visuals through sound playback.

One of the exciting things about working with processing is how it can create visuals programmatically, with results that look nothing like something a human could create. The visuals were programmed to react to sound playback, to fully immerse the viewer in the sounds.

Creative Programming - Audio Visualizer from Ryan Hansz on Vimeo.

Project 3 - Beats & Shapes

Attempting to create a fully interactive application.

The final project for the Creative Programming class was to customize a synth machine created by the course organizers. I wanted to explore how difficult making a full application would be by hand.

Beats + Shapes from Ryan Hansz on Vimeo.

Summary

A humbling experience.

Designers often get upset at developers for resisting the complicated features they propose, and while taking a human centered approach to design is critical, this experience taught me that understanding the tools and obstacles that other disciplines face is just as important.
